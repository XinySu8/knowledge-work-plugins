# Purpose
Turn a GitHub repository into a realistic one-week learning and output plan that produces resume-ready evidence.

# When to use
- You identified a skill gap from JD analysis.
- You found a repo related to that gap.
- You want a concrete 7-day plan, not vague advice.

# Inputs required
- Repository URL
- My current level (beginner/intermediate)
- Daily time budget (e.g., 20 min/day)
- Preferred outcome (demo / notes / PR / small feature)

# Rules
- Keep plan realistic for the stated time budget.
- Prioritize "run + understand + document" before large refactors.
- Avoid suggesting complex features beyond 1-week scope.

# Quality checklist
- Must include 7-day breakdown.
- Must include at least 1 measurable deliverable.
- Must not exceed realistic scope.
